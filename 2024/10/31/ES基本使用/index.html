#ES核心概念

集群(Cluster)

Elasticsearch 集群是由一个或多个节点组成的集合，这些节点共同工作并共享数据。每个集群都有一个唯一的名称，默认为 “elasticsearch“。集群中的每个节点都有一个 ID，用于唯一标识该节点。

节点(Node)

节点是集群中的一个服务器，它存储数据并参与集群的索引和搜索功能。每个节点有一个唯一的名字，可以通过配置文件进行设置。节点可以是主节点、数据节点、协调节点等，不同类型的节点承担不同的职责。一个集群至少有一个节点

索引(Index)

索引是一个或多个文档的集合，这些文档具有相似的特性。每个索引都有一个唯一的名字，索引名称用于索引、搜索、更新和删除操作。索引类似于关系数据库中的表。

文档(Document)

文档是可以被索引的基本单位，以 JSON 格式表示。每个文档是一个 JSON 对象，包含一个或多个字段，每个字段有相应的数据类型（如字符串、数值、日期等）。文档类似于关系数据库中的行。

类型(Type,新版已弃用)

在早期版本的 Elasticsearch 中，类型是索引中的一种逻辑分类，用于将相似文档分组。一个索引可以有多个类型，每个类型包含多个文档。然而，在最新版本中，类型已经被弃用，推荐使用单一类型索引。

##分片 (Shard)

分片是索引数据的一个分片，可以是主分片（Primary Shard）或副本分片（Replica Shard）。分片使得数据可以分布在集群中的多个节点上，从而提升性能和可扩展性。默认情况下，每个索引有 5 个主分片和 1 个副本分片，但可以根据需要进行配置。

倒排索引：采用了Lucene倒排索引作为底层，这种结构适用于快速的全文搜索。一个索引由文档中所不重复的列表构成，对于每一个词，都有一个包含它的文档列表。例如：

term doc_1 doc_2

##副本 (Replica)

副本是主分片的副本，用于提供高可用性和故障转移。在主分片故障时，副本分片可以承担其职责，从而确保数据不丢失。副本分片还可以用于搜索请求的负载均衡。

##映射 (Mapping)

映射定义了索引中文档的结构，即字段类型和属性。它类似于关系数据库中的模式定义，确定了文档中每个字段的数据类型、索引选项、分析器等。

分析器 (Analyzer)

分析器用于将文本字段处理成词条，供索引和搜索使用。分析器由字符过滤器、分词器和词项过滤器组成。分析器在索引和查询时均可使用，不同分析器会对文本进行不同的处理。

##查询 DSL (Query DSL)

查询 DSL 是 Elasticsearch 提供的用于构建查询请求的 JSON 格式的语法。它支持多种查询类型，如匹配查询、布尔查询、范围查询、聚合查询等，可以组合使用以满足复杂的查询需求。

查看集群健康状态

  • status表示集群健康状态
    • green：集群正常，所有的主分片和副本分片都正常。
    • yellow：集群功能正常，但有副本分片没有分配。数据安全性和冗余性降低，不影响读写操作。
    • red：集群存在问题，有主分片没有分配，可能会影响数据的读写操作。
  • node.tot1a表示集群中所有节点的总数
  • node.data表示集群中数据节点的总数
  • shards所有分片的总数
  • pri表示集群中主分片的总数
  • relo正在进行的分片重新分配的数量
  • init正在初始化的分片数量
  • unassign未分配的分片数量
  • pending_tasks挂起任务的数量
  • max_task_wait_time最大任务等待时间
  • active_shards_percent活跃分片的百分比

花式查询

match

match查询用于全文本搜索，类似于模糊查询。它会对查询词进行分词，然后搜索分词后的每个词。具体来说，它会根据分析器（analyzer）的定义来处理文本，这通常包括将文本转换为小写、去除停用词（如“the”、“and”等）以及对文本进行分词。

布尔值

bool查询是一个复合查询，它允许组合多个查询条件（子查询）。通过bool查询，你可以构建复杂的查询逻辑，包括必须匹配的条件（must）、应该匹配的条件（should）、必须不匹配的条件（must_not）和过滤条件（filter）

  • 必须匹配must，全部为真
  • 应该匹配should，一个为真即可
  • 必须不匹配must_not，全部为假
  • 过滤filter，必须为真

使用过滤器指定年龄范围：

gte大于等于，gt大于

lte小于等于，lt小于

must和filter的区别：

must 子句

  • 影响相关性评分**：must子句中的查询条件会影响文档的相关性评分。文档必须匹配所有的must子句，匹配的文档将被返回，并且它们的相关性评分将根据这些查询条件进行计算和调整。
  • 适用于：需要文档符合特定条件，并且希望这些条件对文档的相关性评分产生影响的情况。

filter 子句

  • 不影响相关性评分：filter子句中的查询条件不影响文档的相关性评分。文档必须匹配所有的filter子句，但这些子句不会对文档的相关性评分进行调整。
  • 适用于：需要文档符合特定条件，但不希望这些条件影响文档的相关性评分的情况。过滤条件通常用于常规过滤、预过滤或缓存，以提高查询性能

使用的是Elasticsearch的Transport客户端

  • Transport客户端: 使用TCP通信，更高效但在最新的Elasticsearch版本中不再推荐使用。
  • REST高级客户端: 使用HTTP通信，更现代化和官方推荐，适合大多数应用场景。

如果你使用Elasticsearch 6.x版本，Transport客户端是可以使用的，但是建议尽早迁移到REST客户端，因为它在未来的版本中会更加稳定和受支持。

批量api

Elasticsearch 的 Transport 客户端进行批量操作（Bulk Operation）。批量操作允许你在一次请求中执行多个索引、更新或删除操作。这可以提高性能，因为减少了与服务器的往返次数。
